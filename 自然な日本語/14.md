# 14. メモリAPI（Memory API）

この章では、UNIX/Cプログラムにおけるメモリ割り当てのインタフェースについて説明する。

## 14.1 メモリの2つの種類

### スタックメモリ

コンパイラが暗黙的に管理する。関数内で変数を宣言すると、コンパイラが自動的にスタック上にスペースを確保し、関数終了時に解放する。**自動メモリ**とも呼ばれる。

```c
void func() {
    int x; // スタック上に確保される
    ...
}
```

関数の呼び出しを超えて残す必要があるデータには不向き。

### ヒープメモリ

プログラマが明示的に割り当て・解放する。長期間保持するデータに使う。

```c
void func() {
    int *x = (int *) malloc(sizeof(int));
    ...
}
```

この1行では**スタックとヒープ両方の割り当て**が行われている。ポインタ`x`自体はスタック上に、`malloc()`が確保した領域はヒープ上に配置される。

## 14.2 `malloc()`

ヒープ上にメモリを確保する関数。

```c
#include <stdlib.h>
void *malloc(size_t size);
```

- `size`バイトのメモリを確保し、ポインタを返す（失敗時はNULL）
- `sizeof()`演算子でデータ型のサイズを指定するのが一般的
- 戻り値は`void *`型なので、必要に応じてキャストする

**注意**: `sizeof()`はコンパイル時に評価される演算子であり、関数呼び出しではない。

```c
double *d = (double *) malloc(sizeof(double));
```

文字列の場合は`strlen(s) + 1`でサイズを指定する（末尾のヌル文字のため）。

## 14.3 `free()`

`malloc()`で確保したメモリを解放する。サイズは`malloc()`のライブラリが内部的に追跡しているため、ポインタだけを渡せばよい。

```c
int *x = malloc(10 * sizeof(int));
...
free(x);
```

## 14.4 よくあるエラー

### メモリの確保忘れ

```c
char *src = "hello";
char *dst;           // 未確保！
strcpy(dst, src);    // セグメンテーションフォルト
```

正しくは：
```c
char *dst = (char *) malloc(strlen(src) + 1);
strcpy(dst, src);
```

### メモリ不足の確保（バッファオーバーフロー）

```c
char *dst = (char *) malloc(strlen(src)); // 1バイト足りない！
strcpy(dst, src);
```

たまたま動くこともあるが、セキュリティ脆弱性の原因になる。

### 初期化忘れ

`malloc()`で確保したメモリの値は不定。初期化せずに読むと、予測不能な動作を引き起こす。

### メモリリーク

`free()`の呼び忘れ。短命なプログラムならOSがプロセス終了時に回収するが、長時間稼働するサーバなどでは深刻な問題になる。

### 使用中のメモリの解放（ダングリングポインタ）

まだ使っているメモリを先に解放してしまう。後でアクセスするとクラッシュや不正な動作を引き起こす。

### 二重解放（ダブルフリー）

同じメモリを2回`free()`する。結果は未定義で、クラッシュの原因になる。

### `free()`の不正な呼び出し

`malloc()`から返されたポインタ以外を渡してはならない。

## 14.5 OSのサポート

`malloc()`と`free()`はシステムコールではなく、**ライブラリ呼び出し**だ。内部では以下のシステムコールを使う。

- **`brk`/`sbrk`**: ヒープの終端位置を変更し、ヒープサイズを増減する
- **`mmap()`**: 匿名メモリ領域を作成する。ヒープのように扱える

これらのシステムコールを直接呼ぶのではなく、`malloc()`と`free()`を使うべき。

## 14.6 その他の関数

- **`calloc()`**: メモリを確保し、ゼロで初期化して返す
- **`realloc()`**: 確保済みのメモリ領域のサイズを変更する

## 14.7 まとめ

メモリ管理はCプログラミングにおける重要なスキル。PurifyやValgrindなどのツールを活用してメモリ関連のバグを検出しよう。
