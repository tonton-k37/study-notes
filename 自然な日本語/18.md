# 18. ページング入門（Paging: Introduction）

セグメンテーションは可変サイズの断片に分割するため、外部断片化が避けられない。もう1つのアプローチが、アドレス空間を**固定サイズの単位（ページ）**に分割する**ページング**だ。

## 18.1 基本概念

アドレス空間を固定サイズのページに分割し、物理メモリを同じサイズの**ページフレーム**に分割する。

![](../18/img/fig18_1.PNG)

![](../18/img/fig18_2.PNG)

### ページングの利点

1. **柔軟性**: ヒープやスタックの成長方向を気にせず、アドレス空間を効果的に抽象化できる
2. **空き領域管理の単純さ**: 空きフレームのフリーリストから必要な数だけ取得するだけ

### ページテーブル

各プロセスに、仮想ページから物理フレームへのマッピングを記録する**ページテーブル**を持つ。例えば、(VP0→PF3), (VP1→PF7), (VP2→PF5), (VP3→PF2)など。

### アドレス変換

仮想アドレスを**仮想ページ番号（VPN）**と**オフセット**に分割する。

![](../18/img/fig18_2_1.PNG)

![](../18/img/fig18_2_2.PNG)

例：仮想アドレス21（バイナリ: 010101）の場合
- VPN = 01（ページ1）、オフセット = 0101（5バイト目）
- ページテーブルでVPN 1 → PFN 7を検索
- 物理アドレス = 1110101 = 117

![](../18/img/fig18_2_3.PNG)

![](../18/img/fig18_3.PNG)

## 18.2 ページテーブルの格納場所

ページテーブルは非常に大きくなりうる。32ビットアドレス空間、4KBページの場合：
- VPN: 20ビット → 約100万エントリ
- PTE 4バイト × 100万 = **プロセスあたり4MB**

このサイズのため、MMU内のハードウェアではなく**メモリ上**に格納する。

![](../18/img/fig18_4.PNG)

## 18.3 ページテーブルエントリ（PTE）の内容

最も単純な形式は**線形ページテーブル**（配列）で、VPNで索引する。

各PTEに含まれる主要なビット：

| ビット | 役割 |
|---|---|
| **有効ビット** | この変換が有効かどうか（無効ならトラップ発生） |
| **保護ビット** | 読み取り/書き込み/実行の許可 |
| **プレゼントビット** | ページが物理メモリ内かディスク上か |
| **ダーティビット** | ページが変更されたか |
| **リファレンスビット** | ページがアクセスされたか（ページ置換に使用） |

![](../18/img/fig18_5.PNG)

x86のPTEの例。PFNに加えて、P（プレゼント）、R/W（読み書き）、U/S（ユーザ/スーパバイザ）、A（アクセス済み）、D（ダーティ）などのビットが含まれる。

## 18.4 ページングの速度問題

ページングは**遅い**。すべてのメモリ参照で、まずページテーブルからPTEをフェッチし、次に実際のデータをフェッチする必要がある。つまり、**1回の余分なメモリアクセス**が常に発生する。

```c
VPN = (VirtualAddress & VPN_MASK) >> SHIFT
PTEAddr = PageTableBaseRegister + (VPN * sizeof(PTE))

// PTEをメモリからフェッチ
PTE = AccessMemory(PTEAddr)

// 権限チェック
if (PTE.Valid == False)
    RaiseException(SEGMENTATION_FAULT)
else if (CanAccess(PTE.ProtectBits) == False)
    RaiseException(PROTECTION_FAULT)
else
    // 物理アドレスを構成してアクセス
    offset = VirtualAddress & OFFSET_MASK
    PhysAddr = (PTE.PFN << PFN_SHIFT) | offset
    Register = AccessMemory(PhysAddr)
```

## 18.5 メモリトレースの例

配列の初期化ループを考える：

```c
int array[1000];
for (i = 0; i < 1000; i++)
    array[i] = 0;
```

コンパイル後のアセンブリ：
```
1024 movl $0x0,(%edi,%eax,4)   // 配列要素にゼロを書き込む
1028 incl %eax                 // インデックスをインクリメント
1032 cmpl $0x03e8,%eax         // 1000と比較
1036 jne 0x1024                // 等しくなければループ先頭へ
```

ループ1回あたり**10回のメモリアクセス**が発生する：
- 4回の命令フェッチ + 1回のデータ書き込み + それぞれの5回のページテーブルアクセス

## 18.6 まとめ

ページングはメモリを固定サイズに分割するため、外部断片化が起きない。また非常に柔軟で、アドレス空間の未使用部分にフレームを割り当てなくてよい。

しかし、2つの問題がある：
1. **速度低下**: 余分なメモリアクセスが必要 → 次章のTLBで解決
2. **メモリ浪費**: ページテーブルが巨大 → 次々章の高度なページテーブル構造で解決
