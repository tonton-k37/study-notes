# 15. アドレス変換（Address Translation）

CPU仮想化では「制限付き直接実行（LDE）」を使った。メモリの仮想化でも同様のアプローチをとる。プログラムをハードウェア上で直接実行させ、重要な時点でOSが介入する。目標は**効率性**、**制御**、**柔軟性**の3つだ。

この章で使う手法が**ハードウェアベースのアドレス変換**だ。各メモリアクセスで、ハードウェアが仮想アドレスを物理アドレスに変換する。

## 15.1 前提条件

簡単のため、以下を仮定する（後の章で緩和する）。

- アドレス空間は物理メモリに連続して配置される
- アドレス空間のサイズは物理メモリより小さい
- 各アドレス空間は同じサイズ

## 15.2 例

あるプロセスが以下のコードを実行するとする。

```c
void func() {
    int x = 3000;
    x = x + 3; // この行に注目
```

アセンブリでは：
```
128: movl 0x0(%ebx), %eax  ;アドレス15KBから値をロード
132: addl $0x03, %eax      ;3を加算
135: movl %eax, 0x0(%ebx)  ;値を書き戻す
```

![](../15/img/fig15_1.PNG)

プログラムは「自分のアドレス空間が0番地から始まる」と思っているが、実際には物理メモリの別の場所に配置されている。

![](../15/img/fig15_2.PNG)

図の例では、プロセスのアドレス空間は物理アドレス32KBに再配置されている。

## 15.3 動的再配置（ベースと境界）

1950年代後半に導入された手法。CPU内に2つのハードウェアレジスタを設ける。

- **ベースレジスタ**: アドレス空間の物理メモリ上の開始位置
- **境界レジスタ（バウンドレジスタ）**: アドレス空間のサイズ上限

### アドレス変換の仕組み

```
物理アドレス = 仮想アドレス + ベースレジスタ
```

上記の例で、ベースレジスタが32KB（32768）の場合：

- 命令フェッチ（仮想アドレス128）→ 物理アドレス32896
- データロード（仮想アドレス15KB）→ 物理アドレス47KB

### 境界レジスタによる保護

仮想アドレスが境界レジスタの値を超える場合、CPUが例外を発生させてプロセスを停止する。これにより、プロセスが自分のアドレス空間外のメモリにアクセスすることを防ぐ。

### アドレス変換の例

物理アドレス16KBにサイズ4KBのアドレス空間がロードされた場合：

![](../15/img/fig15_2_1.PNG)

仮想アドレスにベースアドレスを加算するだけで物理アドレスが得られる。範囲外のアドレスは例外となる。

## 15.4 必要なハードウェアサポート

![](../15/img/fig15_3.PNG)

| 機能 | 説明 |
|---|---|
| 特権モード | カーネルモードとユーザモードの区別 |
| ベース/境界レジスタ | MMUの一部として各CPUに搭載 |
| アドレス変換 | 仮想アドレス + ベース で物理アドレスを計算 |
| 境界チェック | アドレスが範囲内か検証 |
| 特権命令 | ベース/境界レジスタを変更する命令（カーネルモード専用） |
| 例外生成 | 境界外アクセス時にOSハンドラを呼び出す |

## 15.5 OSの役割

OSが介入する重要な場面は以下の4つ。

### 1. プロセス作成時

メモリ内にアドレス空間のための領域を見つける。**フリーリスト**で空き領域を管理する。

### 2. プロセス終了時

使用していたメモリをフリーリストに戻す。

### 3. コンテキストスイッチ時

現在のプロセスのベース/境界レジスタの値をPCBに保存し、次のプロセスの値を復元する。停止中にアドレス空間を別の物理位置に移動することも可能。

### 4. 例外ハンドラの登録

ブート時に、境界外アクセスなどの例外ハンドラをインストールする。

## 15.6 まとめ

**ベースと境界（動的再配置）** は、シンプルで高速なメモリ仮想化を提供する。

- 仮想アドレスにベースレジスタを加算するだけで変換
- 境界レジスタでアドレスの正当性を検証
- ハードウェアの回路はシンプルで高速

しかし、この手法には**内部断片化**の問題がある。ヒープとスタックの間の使われていない空間が無駄になる。この問題を解決するのが、次の章で学ぶ**セグメンテーション**だ。
